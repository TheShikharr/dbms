Introduction to SQL-2

SQL-2, also known as Advanced SQL, extends the features of SQL-1 by including more sophisticated operations such as joins, subqueries, constraints, and user access control. It is widely used in enterprise-level databases for enhanced querying, better data integrity, and security.

  • SQL-1 focuses on basic data manipulation — SELECT, INSERT, UPDATE, DELETE, Constraints, Keys, Views, and Transaction Controls (e.g., COMMIT, ROLLBACK). etc.
  • SQL-2 includes complex features like JOINs, SET Operator, Nested Queries.

Below are some commonly used SQL-2 commands along with their purpose, syntax, examples, and expected outputs.

Advanced SQL (SQL-2) Commands

JOINS

INNER JOIN

Purpose: Retrieves matching rows from two or more tables.

Syntax:

SELECT columns FROM table1 INNER JOIN table2 ON table1.id = table2.id;

Example:

SELECT students.name, grades.grade
FROM students
INNER JOIN grades ON students.id = grades.student_id;

Output: Shows student names along with their grades.

LEFT JOIN

Purpose: Retrieves all records from the left table, and the matched records from the right table.

Syntax:

SELECT columns FROM table1 LEFT JOIN table2 ON table1.id = table2.id;

Example:

SELECT employees.name, departments.dept_name
FROM employees
LEFT JOIN departments ON employees.dept_id = departments.id;

Output: Lists all employees, even those without departments.

RIGHT JOIN

Purpose: Retrieves all records from the right table, and the matched records from the left table.

Syntax:

SELECT columns FROM table1 RIGHT JOIN table2 ON table1.id = table2.id;

Example:

SELECT employees.name, departments.dept_name
FROM employees
RIGHT JOIN departments ON employees.dept_id = departments.id;

Output: Lists all departments, even those without employees.

FULL OUTER JOIN

Purpose: Retrieves all records when there is a match in either left or right table.

Syntax:

SELECT columns FROM table1 FULL OUTER JOIN table2 ON table1.id = table2.id;

Example:

SELECT students.name, grades.grade
FROM students
FULL OUTER JOIN grades ON students.id = grades.student_id;

Output: Displays all students and grades, including unmatched records from either table.

SELF JOIN

Purpose: Joins a table with itself to compare rows within the same table.

Syntax:

SELECT A.column_name, B.column_name
FROM table_name A, table_name B
WHERE condition;

Example:

SELECT A.name AS Employee1, B.name AS Employee2
FROM employees A, employees B
WHERE A.manager_id = B.id;

Output: Displays employees along with their managers.

CROSS JOIN

Purpose: Returns the Cartesian product of two tables.

Syntax:

SELECT * FROM table1 CROSS JOIN table2;

Example:

SELECT employees.name, projects.name
FROM employees
CROSS JOIN projects;

Output: Lists all possible combinations of employees and projects.

NATURAL JOIN

Purpose: Automatically joins tables based on columns with the same names and data types in both tables.

Syntax:

SELECT * FROM table1 NATURAL JOIN table2;

Example:

SELECT *
FROM students
NATURAL JOIN enrollments;

Output: Shows combined rows where common columns match.

EQUI JOIN

Purpose: Combines rows from two or more tables using equality comparison in the JOIN condition.

Syntax:

SELECT columns FROM table1, table2 WHERE table1.id = table2.id;

Example:

SELECT employees.name, departments.dept_name
FROM employees, departments
WHERE employees.dept_id = departments.id;

Output: Shows employees and their corresponding departments.

CARTESIAN JOIN

Purpose: Returns the Cartesian product of rows from the tables (same as CROSS JOIN).

Syntax:

SELECT * FROM table1, table2;

Example:

SELECT students.name, courses.title
FROM students, courses;

Output: Lists all possible combinations of students and courses.

SET Operator

UNION

Purpose: Combines the result sets of two or more SELECT statements (removes duplicates).

Syntax:

SELECT column_name FROM table1
UNION
SELECT column_name FROM table2;

Example:

SELECT city FROM customers
UNION
SELECT city FROM suppliers;

Output: Returns a distinct list of cities from both customers and suppliers.

UNION ALL

Purpose: Combines the result sets of two SELECT statements including duplicates.

Syntax:

SELECT column_name FROM table1
UNION ALL
SELECT column_name FROM table2;

Example:

SELECT city FROM customers
UNION ALL
SELECT city FROM suppliers;

Output: Returns all cities from both tables, including duplicates.

INTERSECT

Purpose: Returns common records from two SELECT queries.

Syntax:

SELECT column_name FROM table1
INTERSECT
SELECT column_name FROM table2;

Example:

SELECT city FROM customers
INTERSECT
SELECT city FROM suppliers;

Output: Returns cities that exist in both customers and suppliers tables.

MINUS (EXCEPT in some databases)

Purpose: Returns records from the first SELECT query that are not present in the second.

Syntax:

SELECT column_name FROM table1
MINUS
SELECT column_name FROM table2;

Example:

SELECT city FROM customers
MINUS
SELECT city FROM suppliers;

Output: Returns cities that are in customers but not in suppliers.

Nested Queries

NESTED QUERY 1 - IN Clause

Purpose: Select records from a table where a column's value matches any value in a subquery result.

SELECT name FROM students
WHERE id IN (SELECT student_id FROM enrollments WHERE course_id = 101);

Output: List of students enrolled in course ID 101.

NESTED QUERY 2 - EXISTS Clause

Purpose: Select records where a related subquery returns results.

SELECT name FROM students s
WHERE EXISTS (SELECT * FROM enrollments e WHERE e.student_id = s.id AND e.course_id = 101);

Output: Students who are enrolled in course ID 101.

NESTED QUERY 3 - ANY Keyword

Purpose: Compare a value to any value returned by a subquery.

SELECT name, salary FROM employees
WHERE salary > ANY (SELECT salary FROM employees WHERE department = 'HR');

Output: Employees who earn more than at least one HR employee.

NESTED QUERY 4 - ALL Keyword

Purpose: Compare a value to all values returned by a subquery.

SELECT name, salary FROM employees
WHERE salary > ALL (SELECT salary FROM employees WHERE department = 'HR');

Output: Employees who earn more than every HR employee.

NESTED QUERY 5 - FROM Clause Subquery

Purpose: Use a subquery as a derived table in the FROM clause.

SELECT avg_salary.department, avg_salary.avg_sal
FROM (SELECT department, AVG(salary) AS avg_sal FROM employees GROUP BY department) AS avg_salary
WHERE avg_salary.avg_sal > 50000;

Output: Departments with average salary greater than 50,000.

NESTED QUERY 6 - SELECT in SELECT

Purpose: Return data using a subquery in the SELECT clause.

SELECT name, (SELECT dept_name FROM departments WHERE departments.id = employees.dept_id) AS department
FROM employees;

Output: List of employees with their department names using subquery in SELECT.

NESTED QUERY 7 - Correlated Subquery

Purpose: Use subquery that references column from the outer query.

SELECT name, salary FROM employees e1
WHERE salary > (SELECT AVG(salary) FROM employees e2 WHERE e1.department = e2.department);

Output: Employees who earn more than the average salary in their own department.

NESTED QUERY 8 - Subquery in UPDATE

Purpose: Use subquery to set new values based on another table.

UPDATE employees
SET salary = salary * 1.1
WHERE department = (SELECT id FROM departments WHERE dept_name = 'Sales');

Output: Increases salary by 10% for employees in the Sales department.
