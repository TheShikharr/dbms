PL/SQL Introduction

PL/SQL (Procedural Language/Structured Query Language) is Oracle Corporation’s procedural extension for SQL and the Oracle relational database. PL/SQL allows developers to write code in a procedural format, using control structures like loops and conditions along with SQL.

Advantages of PL/SQL

  • Tight Integration with SQL: Combines SQL’s data manipulation power with the processing power of procedural languages.
  • Improved Performance: Blocks of code are sent to the database at once, reducing network traffic.
  • Modular Programming: Allows the use of procedures, functions, packages, and triggers.
  • Robust Error Handling: Built-in exception handling helps manage runtime errors effectively.
  • Security: Supports user privileges and secure program unit execution.

Disadvantages of PL/SQL

  • Limited Portability: PL/SQL is Oracle-specific and not compatible with other database systems like MySQL or SQL Server.
  • Learning Curve: Requires learning both SQL and procedural concepts.
  • GUI Limitations: Lacks built-in GUI support; requires integration with other tools for UI development.
  • Resource Intensive: Complex PL/SQL code can consume more resources if not optimized properly.

PL/SQL Architecture

PL/SQL engine executes procedural statements and sends SQL commands to the SQL engine. It supports:

  • PL/SQL Block Structure
  • Control Structures
  • Exception Handling

PL/SQL Fundamentals

  • PL/SQL blocks (Anonymous, Named)
  • Control statements (IF, LOOP, WHILE, FOR)
  • Cursors and Exceptions
  • Stored procedures and functions

Data Types in PL/SQL

  • Scalar: NUMBER, VARCHAR2, DATE, BOOLEAN
  • Composite: RECORD, TABLE
  • Reference: REF CURSOR

Block Structure

Purpose: Defines the structure of every PL/SQL program unit.

Syntax:

DECLARE
  -- variable declarations
BEGIN
  -- executable statements
EXCEPTION
  -- exception handlers
END;

Example:

DECLARE
  v_message VARCHAR2(50);
BEGIN
  v_message := 'Hello from PL/SQL';
  DBMS_OUTPUT.PUT_LINE(v_message);
EXCEPTION
  WHEN OTHERS THEN
    DBMS_OUTPUT.PUT_LINE('Error occurred');
END;
Output: Hello from PL/SQL

Declaring Variables

Purpose: Used to store data values temporarily during PL/SQL block execution.

Syntax:

  variable_name datatype [ := initial_value ];

Example:

  DECLARE
    v_name VARCHAR2(50);
    v_total NUMBER := 0;
    v_today DATE := SYSDATE;
  BEGIN
    DBMS_OUTPUT.PUT_LINE('Today is: ' || v_today);
  END;
Output: Today is: 10-APR-2025

Variable Scope & Visibility

Purpose: Controls where variables can be accessed within nested blocks.

Explanation:

  • Outer block variables can be accessed by inner blocks.
  • Inner block variables cannot be accessed by outer blocks.

Example:

DECLARE
  v_outer NUMBER := 10;
BEGIN
  DECLARE
    v_inner NUMBER := 20;
  BEGIN
    DBMS_OUTPUT.PUT_LINE('Outer: ' || v_outer);
    DBMS_OUTPUT.PUT_LINE('Inner: ' || v_inner);
  END;
  -- DBMS_OUTPUT.PUT_LINE(v_inner); -- Error: not accessible here
END;
Output: Outer: 10 Inner: 20

Assignments and Expressions

Purpose: Used to assign values to variables and perform calculations or string manipulations.

Syntax:

variable_name := expression;

Example:

DECLARE
  a NUMBER := 10;
  b NUMBER := 5;
  result NUMBER;
BEGIN
  result := a + b;
  DBMS_OUTPUT.PUT_LINE('Result: ' || result);
END;
Output: Result: 15

%type & %rowtype

1. %TYPE Attribute

Purpose: Declares a variable with the same data type as a column in a table.

Syntax:

declared_variable table_name.column_name%TYPE;

Example:

DECLARE
  v_name students.name%TYPE;
BEGIN
  SELECT name INTO v_name FROM students WHERE student_id = 1;
  DBMS_OUTPUT.PUT_LINE('Student Name: ' || v_name);
END;
Output: Student Name: Rahul

2. %ROWTYPE Attribute

Purpose: Declares a variable with the structure of an entire row of a table.

Syntax:

declared_record table_name%ROWTYPE;

Example:

DECLARE
  student_record students%ROWTYPE;
BEGIN
  SELECT * INTO student_record FROM students WHERE student_id = 1;
  DBMS_OUTPUT.PUT_LINE('ID: ' || student_record.student_id);
  DBMS_OUTPUT.PUT_LINE('Name: ' || student_record.name);
END;
Output: ID: 1 Name: Rahul

Control Structures

1. Conditional Control

Purpose: To control the flow of execution based on conditions.

IF-THEN-END IF

Syntax:

IF condition THEN
  -- statements
END IF;

Example:

DECLARE
  salary NUMBER := 6000;
BEGIN
  IF salary > 5000 THEN
    DBMS_OUTPUT.PUT_LINE('Salary is above 5000');
  END IF;
END;
Output: Salary is above 5000

IF-THEN-ELSE-END IF

Syntax:

IF condition THEN
  -- statements
ELSE
  -- statements
END IF;

Example:

DECLARE
  salary NUMBER := 4000;
BEGIN
  IF salary > 5000 THEN
    DBMS_OUTPUT.PUT_LINE('High Salary');
  ELSE
    DBMS_OUTPUT.PUT_LINE('Low Salary');
  END IF;
END;
Output: Low Salary

IF-THEN-ELSIF-END IF

Syntax:

IF condition1 THEN
  -- statements
ELSIF condition2 THEN
  -- statements
ELSE
  -- statements
END IF;

Example:

DECLARE
  marks NUMBER := 88;
BEGIN
  IF marks >= 90 THEN
    DBMS_OUTPUT.PUT_LINE('Grade A');
  ELSIF marks >= 75 THEN
    DBMS_OUTPUT.PUT_LINE('Grade B');
  ELSE
    DBMS_OUTPUT.PUT_LINE('Grade C');
  END IF;
END;
Output: Grade B

2. CASE Expression

Purpose: Used to implement conditional logic in expressions.

Syntax:

        CASE selector
        WHEN 'value1' THEN result1
        WHEN 'value2' THEN result2
        ...
        ELSE resultN
      END CASE;

Example:

  DECLARE
    grade CHAR(1) := 'B';
  BEGIN
    CASE grade
      WHEN 'A' THEN DBMS_OUTPUT.PUT_LINE('Excellent');
      WHEN 'B' THEN DBMS_OUTPUT.PUT_LINE('Very Good');
      WHEN 'C' THEN DBMS_OUTPUT.PUT_LINE('Good');
      ELSE DBMS_OUTPUT.PUT_LINE('Needs Improvement');
    END CASE;
  END;
Output: Very Good

3. Loop/Iterative Control

Simple LOOP

Purpose: Repeats a block of statements an indefinite number of times.

Syntax:

        LOOP
  statements;
  EXIT WHEN condition;
END LOOP;

Example:

        DECLARE
          i NUMBER := 1;
        BEGIN
          LOOP
            EXIT WHEN i > 5;
            DBMS_OUTPUT.PUT_LINE('Iteration: ' || i);
            i := i + 1;
          END LOOP;
        END;
Output: 
            Iteration: 1 
            Iteration: 2
            Iteration: 3
            Iteration: 4
            Iteration: 5

WHILE LOOP

Purpose: Repeats a block of statements while a condition is true.

Syntax:

        WHILE condition LOOP
  statements;
END LOOP;

Example:

  DECLARE
    i NUMBER := 1;
  BEGIN
    WHILE i <= 3 LOOP
      DBMS_OUTPUT.PUT_LINE('Count: ' || i);
      i := i + 1;
    END LOOP;
  END;
Output:
  Count: 1
  Count: 2
  Count: 3

FOR LOOP

Purpose: Iterates over a specific range.

Syntax:

        FOR counter IN [REVERSE] lower_bound..upper_bound LOOP
        statements;
      END LOOP;

Example:

  BEGIN
    FOR i IN 1..3 LOOP
      DBMS_OUTPUT.PUT_LINE('Step: ' || i);
    END LOOP;
  END;
Output:
  Step: 1
  Step: 2
  Step: 3

4. Squential Control

GOTO Statement

Purpose: Transfers control to a labeled statement.

Syntax:

          GOTO label_name;
          <>
            statements;

Example:

  DECLARE
    x NUMBER := 1;
  BEGIN
    IF x = 1 THEN
      GOTO jump_here;
    END IF;
    DBMS_OUTPUT.PUT_LINE('This will be skipped');
    <>
    DBMS_OUTPUT.PUT_LINE('Landed after GOTO');
  END;
Output: Landed after GOTO

NULL Statement

Purpose: Represents a no-operation placeholder in control structures.

Syntax:

          NULL;

Example:

  DECLARE
    flag BOOLEAN := TRUE;
  BEGIN
    IF flag THEN
      NULL; -- do nothing
    ELSE
      DBMS_OUTPUT.PUT_LINE('This will not run');
    END IF;
    DBMS_OUTPUT.PUT_LINE('Done');
  END;
Output: Done

Cursor

Purpose: A cursor is a pointer that allows row-by-row processing of the result set.

1. Implicit Cursor

Purpose: Automatically created by Oracle when an SQL statement like INSERT, UPDATE, DELETE or SELECT INTO is executed.

Syntax:

DECLARE
  variable_name datatype;
BEGIN
  SELECT column INTO variable_name FROM table_name WHERE condition;
END;

Example:

        DECLARE
          v_total_students NUMBER;
        BEGIN
          SELECT COUNT(*) INTO v_total_students FROM students;
          DBMS_OUTPUT.PUT_LINE('Total Students: ' || v_total_students);
        END;
Output: Total Students: 8

2. Explicit Cursor

Purpose: Declared by the programmer for queries that return more than one row.

Syntax:

DECLARE
  CURSOR cursor_name IS select_statement;
  variable datatype;
BEGIN
  OPEN cursor_name;
  LOOP
    FETCH cursor_name INTO variable;
    EXIT WHEN cursor_name%NOTFOUND;
    -- Processing logic
  END LOOP;
  CLOSE cursor_name;
END;

Example:

        DECLARE
          CURSOR cur_students IS SELECT name FROM students;
          v_name students.name%TYPE;
        BEGIN
          OPEN cur_students;
          LOOP
            FETCH cur_students INTO v_name;
            EXIT WHEN cur_students%NOTFOUND;
            DBMS_OUTPUT.PUT_LINE('Student Name: ' || v_name);
          END LOOP;
          CLOSE cur_students;
        END;
Output: Student Name: Aman Student Name: Rahul Student Name: Priya ...

Exception Handling

Purpose: To handle errors that occur during the execution of a PL/SQL block.

Syntax:

BEGIN
  -- Code that might raise an exception
EXCEPTION
  WHEN exception_name THEN
    -- Exception handling code
END;

Example:

BEGIN
  DBMS_OUTPUT.PUT_LINE(10/0);
EXCEPTION
  WHEN ZERO_DIVIDE THEN
    DBMS_OUTPUT.PUT_LINE('Error: Division by zero not allowed');
END;
Output: Error: Division by zero not allowed

1. User-Defined Exception

Purpose: Allows the programmer to define their own exception based on specific business logic.

Syntax:

DECLARE
  exception_name EXCEPTION;
BEGIN
  -- Some logic
  RAISE exception_name;
EXCEPTION
  WHEN exception_name THEN
    -- Handler logic
END;

Example:

DECLARE
  insufficient_salary EXCEPTION;
  salary NUMBER := 1000;
BEGIN
  IF salary < 5000 THEN
    RAISE insufficient_salary;
  END IF;
EXCEPTION
  WHEN insufficient_salary THEN
    DBMS_OUTPUT.PUT_LINE('Error: Salary below minimum threshold');
END;
Output: Error: Salary below minimum threshold

2. Predefined Exception

Purpose: Built-in exceptions that are automatically raised during PL/SQL execution.

Syntax:

BEGIN
  -- Statements that may raise predefined exceptions
EXCEPTION
  WHEN predefined_exception THEN
    -- Handler logic
END;

Example:

DECLARE
  v_name VARCHAR2(50);
BEGIN
  SELECT name INTO v_name FROM students WHERE student_id = 1000;
  DBMS_OUTPUT.PUT_LINE('Student Name: ' || v_name);
EXCEPTION
  WHEN NO_DATA_FOUND THEN
    DBMS_OUTPUT.PUT_LINE('No such student exists');
END;
Output: No such student exists

Triggers

Purpose: A trigger is a stored procedure that automatically runs when an event occurs in the database.

Syntax:

CREATE [OR REPLACE] TRIGGER trigger_name
{BEFORE | AFTER | INSTEAD OF} 
{INSERT | DELETE | UPDATE [OF column_name]} 
ON table_name 
[REFERENCING OLD AS old NEW AS new] 
[FOR EACH ROW] 
[WHEN (condition)]
DECLARE
  -- variable declarations
BEGIN
  -- trigger logic
EXCEPTION
  -- exception handling
END;

1. Statement-Level Trigger

Purpose: Executes once for the triggering event, regardless of the number of rows affected.

Syntax:

CREATE OR REPLACE TRIGGER trigger_name
BEFORE INSERT ON table_name
BEGIN
  -- Trigger logic
END;

Example:

CREATE OR REPLACE TRIGGER trg_before_insert
BEFORE INSERT ON employees
BEGIN
  DBMS_OUTPUT.PUT_LINE('Insert operation triggered');
END;
Output: Insert operation triggered

2. Row-Level Trigger

Purpose: Executes once for each row affected by the triggering event.

Syntax:

CREATE OR REPLACE TRIGGER trigger_name
BEFORE INSERT OR UPDATE ON table_name
FOR EACH ROW
BEGIN
  -- Row-level logic using :NEW and :OLD
END;

Example:

CREATE OR REPLACE TRIGGER trg_salary_check
BEFORE INSERT OR UPDATE ON employees
FOR EACH ROW
BEGIN
  IF :NEW.salary < 0 THEN
    RAISE_APPLICATION_ERROR(-20001, 'Salary cannot be negative');
  END IF;
END;
Output: Error raised if inserted/updated salary is negative

3. BEFORE Trigger

Purpose: Executes the trigger action before the triggering DML event occurs.

Syntax:

CREATE OR REPLACE TRIGGER trigger_name
BEFORE INSERT OR UPDATE ON table_name
FOR EACH ROW
BEGIN
  -- Action before insert or update
END;

Example:

CREATE OR REPLACE TRIGGER trg_check_name
BEFORE INSERT ON students
FOR EACH ROW
BEGIN
  IF :NEW.name IS NULL THEN
    RAISE_APPLICATION_ERROR(-20001, 'Name cannot be null');
  END IF;
END;
Output: Raises an error if the student's name is null before inserting.

4. AFTER Trigger

Purpose: Executes the trigger action after the triggering DML event has occurred.

Syntax:

CREATE OR REPLACE TRIGGER trigger_name
AFTER INSERT OR DELETE ON table_name
FOR EACH ROW
BEGIN
  -- Action after insert or delete
END;

Example:

CREATE OR REPLACE TRIGGER trg_log_insert
AFTER INSERT ON employees
FOR EACH ROW
BEGIN
  INSERT INTO log_table VALUES(:NEW.id, SYSTIMESTAMP);
END;
Output: Logs inserted employee ID and timestamp after each insert.

5. INSTEAD OF Trigger

Purpose: Executes instead of the triggering DML statement, typically used with views.

Syntax:

CREATE OR REPLACE TRIGGER trigger_name
INSTEAD OF INSERT ON view_name
BEGIN
  -- Custom logic to handle insert on view
END;

Example:

CREATE OR REPLACE TRIGGER trg_view_insert
INSTEAD OF INSERT ON student_view
BEGIN
  INSERT INTO students(id, name) VALUES(:NEW.id, :NEW.name);
END;
Output: Performs a custom insert into the base table when inserting into a view.
