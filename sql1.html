Introduction to SQL-1

SQL-1 refers to the basic set of SQL (Structured Query Language) commands used to interact with databases. These commands allow users to select, filter, sort, and limit the data stored in tables.

SQL-1 is typically the first level of SQL that beginners learn, as it covers the fundamental skills needed to retrieve data from relational databases.

Common SQL-1 Commands include:

  • SELECT – Retrieve data from one or more columns
  • FROM – Specify the table
  • WHERE – Filter rows based on conditions
  • ORDER BY – Sort the results
  • LIMIT – Limit the number of results

Below are examples of these commands in action, with output displayed as cards for better visualization.

SQL-1 Commands

DDL Commands

CREATE TABLE

Purpose: Create a new table in the database

Syntax:

CREATE TABLE table_name (column1 datatype, column2 datatype);

Example:

CREATE TABLE students (id INT, name TEXT, age INT);

Output: Table created

DROP TABLE

Purpose: Delete an existing table and its data

Syntax:

DROP TABLE table_name;

Example:

DROP TABLE students;

Output: Table dropped

ALTER TABLE

Purpose: Modify an existing table (e.g., add or delete a column)

Syntax:

ALTER TABLE table_name ADD column_name datatype;
ALTER TABLE table_name DROP COLUMN column_name;

Example:

ALTER TABLE students ADD email TEXT;

Output: Column added to table

TRUNCATE

Purpose: Removes all rows from a table, but the table structure remains

Syntax:

TRUNCATE TABLE table_name;

Example:

TRUNCATE TABLE students;

Output: All data in the students table is deleted, but the table still exists

RENAME

Purpose: Changes the name of a table

Syntax:

ALTER TABLE old_table_name RENAME TO new_table_name;

Example:

ALTER TABLE students RENAME TO learners;

Output: The table 'students' is renamed to 'learners'

DML Commands

SELECT

Purpose: Retrieve data from a table

Syntax:

SELECT column1, column2 FROM table_name;

Example:

SELECT name, age FROM students;

Output: List of student names and ages

INSERT

Purpose: Insert new data into a table

Syntax:

INSERT INTO table_name (column1, column2) VALUES (value1, value2);

Example:

INSERT INTO students (name, age) VALUES ('Alice', 22);

Output: 1 row inserted

UPDATE

Purpose: Update existing data in a table

Syntax:

UPDATE table_name SET column1 = value1 WHERE condition;

Example:

UPDATE students SET age = 23 WHERE name = 'Alice';

Output: 1 row updated

DELETE

Purpose: Delete data from a table

Syntax:

DELETE FROM table_name WHERE condition;

Example:

DELETE FROM students WHERE name = 'Alice';

Output: 1 row deleted

DCL Commands

GRANT

Purpose: Provides privileges to users

Syntax:

GRANT privilege ON object TO user;

Example:

GRANT SELECT ON students TO user1;

Output: user1 can now select data from students table

REVOKE

Purpose: Removes privileges from users

Syntax:

REVOKE privilege ON object FROM user;

Example:

REVOKE SELECT ON students FROM user1;

Output: user1 can no longer select data from students table

TCL Commands

COMMIT

Purpose: Saves all changes made in the current transaction

Syntax:

COMMIT;

Example:

INSERT INTO students VALUES (6, 'Ravi', 22);
COMMIT;

Output: Inserted data is permanently saved

ROLLBACK

Purpose: Undoes changes made in the current transaction

Syntax:

ROLLBACK;

Example:

DELETE FROM students WHERE id = 6;
ROLLBACK;

Output: Delete action is undone, data is restored

Constraints

NOT NULL Constraint

Purpose: Ensures that a column cannot have NULL values.

CREATE TABLE products (
  product_id INT,
  name VARCHAR(100) NOT NULL
);

Output: Ensures every product must have a name.

UNIQUE Constraint

Purpose: Ensures that all values in a column are different.

CREATE TABLE customers (
  customer_id INT,
  phone_number VARCHAR(15) UNIQUE
);

Output: Prevents duplicate phone numbers.

PRIMARY KEY Constraint

Purpose: Combines NOT NULL and UNIQUE to uniquely identify a row.

CREATE TABLE orders (
  order_id INT PRIMARY KEY,
  order_date DATE
);

Output: Uniquely identifies each order.

FOREIGN KEY Constraint

Purpose: Links two tables by enforcing referential integrity.

CREATE TABLE payments (
  payment_id INT,
  order_id INT,
  FOREIGN KEY (order_id) REFERENCES orders(order_id)
);

Output: Ensures order_id exists in the orders table.

CHECK Constraint

Purpose: Ensures values meet a specific condition.

CREATE TABLE employees (
  emp_id INT,
  salary DECIMAL(10, 2),
  CHECK (salary > 0)
);

Output: Ensures salary is a positive number.

DEFAULT Constraint

Purpose: Assigns a default value if none is provided.

CREATE TABLE accounts (
  id INT,
  status VARCHAR(10) DEFAULT 'active'
);

Output: New accounts get 'active' as the default status.

Keywords

DISTINCT Keyword

Purpose: Removes duplicate values from the result set.

SELECT DISTINCT department FROM employees;

Output: List of unique departments.

BETWEEN Keyword

Purpose: Filters results within a specified range.

SELECT * FROM products WHERE price BETWEEN 100 AND 500;

Output: Products priced between 100 and 500.

IN Keyword

Purpose: Matches any value within a list.

SELECT * FROM employees WHERE department IN ('HR', 'Sales');

Output: Employees in HR or Sales.

IS NULL Keyword

Purpose: Checks if a column has NULL value.

SELECT * FROM employees WHERE manager_id IS NULL;

Output: Lists employees who do not have a manager assigned.

IS NOT NULL Keyword

Purpose: Checks if a column does not have NULL value.

SELECT * FROM employees WHERE manager_id IS NOT NULL;

Output: Lists employees who have a manager assigned.

Clauses

WHERE

Purpose: Filters records that meet a specific condition

Syntax:

SELECT column1 FROM table_name WHERE condition;

Example:

SELECT * FROM students WHERE age > 20;

Output: Students older than 20

GROUP BY

Purpose: Groups rows that have the same values into summary rows

Syntax:

SELECT column, COUNT(*) FROM table_name GROUP BY column;

Example:

SELECT age, COUNT(*) FROM students GROUP BY age;

Output: Number of students per age

HAVING

Purpose: Filters groups based on a condition (used with GROUP BY)

Syntax:

SELECT column, COUNT(*) FROM table_name GROUP BY column HAVING COUNT(*) > 1;

Example:

SELECT age, COUNT(*) FROM students GROUP BY age HAVING COUNT(*) > 1;

Output: Ages with more than one student

ORDER BY

Purpose: Sorts the result set in ascending or descending order

Syntax:

SELECT column1 FROM table_name ORDER BY column1 [ASC|DESC];

Example:

SELECT name, age FROM students ORDER BY age DESC;

Output: Students sorted by age descending

LIMIT

Purpose: Specifies the number of records to return

Syntax:

SELECT column1 FROM table_name LIMIT number;

Example:

SELECT * FROM students LIMIT 3;

Output: First 3 student records

Views

VIEW

Purpose: Creates a virtual table based on the result set of a SELECT query

Syntax:

CREATE VIEW view_name AS SELECT column1 FROM table_name WHERE condition;

Example:

CREATE VIEW student_view AS SELECT name, age FROM students WHERE age > 18;

Output: A virtual table with students older than 18

DELETE VIEW

Purpose: Deletes an existing view from the database

Syntax:

DROP VIEW view_name;

Example:

DROP VIEW student_view;

Output: The view named 'student_view' is removed from the database

DATE FUNCTION

DATE FUNCTION

Purpose: Used to work with date values in SQL

Common Functions:

CURRENT_DATE       -- returns current date
CURRENT_TIME       -- returns current time
CURRENT_TIMESTAMP  -- returns current date and time
DATE_PART('year', date) -- extracts part of a date

Example:

SELECT CURRENT_DATE;
SELECT DATE_PART('year', '2024-04-06');

Output: Current date and extracted year (e.g., 2024)

Aggregate Functions

COUNT()

Purpose: Returns the number of rows that match a condition

Syntax:

SELECT COUNT(column_name) FROM table_name;

Example:

SELECT COUNT(*) FROM students;

Output: Number of students

SUM()

Purpose: Returns the total sum of a numeric column

Syntax:

SELECT SUM(column_name) FROM table_name;

Example:

SELECT SUM(age) FROM students;

Output: Total of student ages

AVG()

Purpose: Returns the average value of a numeric column

Syntax:

SELECT AVG(column_name) FROM table_name;

Example:

SELECT AVG(age) FROM students;

Output: Average student age

MIN()

Purpose: Returns the smallest value in a column

Syntax:

SELECT MIN(column_name) FROM table_name;

Example:

SELECT MIN(age) FROM students;

Output: Youngest student's age

MAX()

Purpose: Returns the largest value in a column

Syntax:

SELECT MAX(column_name) FROM table_name;

Example:

SELECT MAX(age) FROM students;

Output: Oldest student's age

Wildcard Characters

LIKE Operator

Purpose: Used to search for a specified pattern in a column

Wildcard Characters:

%  - represents zero, one, or multiple characters
_  - represents a single character

Example:

SELECT * FROM students WHERE name LIKE 'A%';
SELECT * FROM students WHERE name LIKE '_a%';

Output: Names that start with 'A' and names where the second letter is 'a'
